## conda란?

**conda**는 Python을 포함한 다양한 언어의 패키지와 실행 환경을 관리할 수 있는  
**환경 관리 도구(Environment Manager)**이다.

venv가 Python의 **가상 환경 생성 도구**라면,  
conda는 **가상 환경 + 패키지 관리자 + Python 버전 관리자**의 역할을 함께 수행한다.

---

### conda가 등장한 배경

Python 생태계는 웹 개발뿐 아니라  
데이터 분석, 머신러닝, 과학 계산 등 다양한 분야에서 사용된다.

이러한 분야에서는 다음과 같은 요구가 많았다.

- Python 버전 자체를 프로젝트별로 다르게 사용
- C/C++ 기반 라이브러리(Numpy, SciPy 등)의 복잡한 의존성
- 운영체제별 빌드 차이로 인한 설치 문제

conda는 이러한 문제를 해결하기 위해  
**Anaconda 프로젝트에서 개발된 환경 관리 도구**이다.

---

### conda의 개념

conda는 다음을 하나의 환경으로 관리한다.

- Python 인터프리터 (버전 포함)
- 패키지와 라이브러리
- 해당 환경에 필요한 의존성

즉, conda 환경은  
**Python 자체까지 포함한 완전한 실행 환경**이다.

이 점이 venv와 가장 큰 차이이다.

---

### conda 환경의 특징

conda로 생성한 환경은 다음과 같은 특성을 가진다.

- 환경마다 Python 버전을 다르게 설정 가능
- 패키지 간 의존성을 conda가 직접 해결
- Python 외의 패키지도 함께 관리 가능

각 conda 환경은 서로 완전히 분리되어 있으며,  
한 환경의 변경이 다른 환경이나 시스템 전역에 영향을 주지 않는다.

---

### conda 환경 생성 개념

conda에서는 보통 다음과 같이 환경을 생성한다.

```bash
conda create -n myenv python=3.10
```

이 명령은:

- `myenv`라는 이름의 환경을 만들고
- Python 3.10을 포함한 환경을 구성한다

---

### conda 환경 활성화 / 비활성화

#### 환경 활성화

```bash
conda activate myenv
```

#### 환경 비활성화

```bash
conda deactivate
```

활성화되면 해당 환경의 Python과 패키지가 사용된다.

---

### conda에서의 패키지 관리

conda 환경이 활성화된 상태에서 패키지를 설치하면  
해당 환경에만 설치된다.

```bash
conda install numpy
```

conda는 패키지를 설치할 때  
**의존성 충돌을 자동으로 계산하고 해결**한다는 특징이 있다.

---

### conda의 특징 정리

- Python 버전까지 포함한 환경 관리
- 패키지 의존성 자동 해결
- Python 외 패키지도 관리 가능
- 데이터 분석 / 머신러닝 분야에서 많이 사용

---

### conda의 한계

conda는 강력하지만 몇 가지 단점도 있다.

- 설치 용량이 크다
- 환경 생성과 패키지 설치 속도가 느릴 수 있다
- 단순한 웹 프로젝트에는 다소 과할 수 있다

이러한 이유로  
가볍고 단순한 환경이 필요한 경우에는 **venv**가 더 적합할 수 있다.
