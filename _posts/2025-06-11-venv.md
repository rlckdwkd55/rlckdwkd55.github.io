## venv란?

**venv**는 Python에서 제공하는 **가상 환경(Virtual Environment)** 생성 도구이다.  
하나의 시스템에서 **여러 Python 프로젝트를 독립적으로 관리**할 수 있도록 도와준다.

Python 프로젝트를 진행하다 보면  
프로젝트마다 사용하는 라이브러리와 버전이 달라지는 경우가 많다.  
venv는 이러한 **의존성 충돌 문제를 해결하기 위한 표준적인 방법**이다.

---

### 가상 환경이 필요한 이유

Python은 기본적으로 **전역 환경(Global Environment)**에 라이브러리를 설치한다.  
이 방식은 다음과 같은 문제를 만든다.

- 프로젝트 A와 B가 서로 다른 라이브러리 버전을 요구
- 한 프로젝트에서 라이브러리를 업그레이드하면 다른 프로젝트가 깨짐
- 어떤 프로젝트가 어떤 라이브러리를 쓰는지 관리가 어려움

이 문제를 해결하기 위해  
**프로젝트별로 독립된 Python 실행 환경**이 필요해졌고,  
그 결과가 가상 환경이다.

---

### venv의 개념

venv는 다음을 포함하는 **독립된 Python 환경**을 생성한다.

- Python 인터프리터
- pip
- 해당 환경에서 설치한 라이브러리들

각 venv 환경은 서로 완전히 분리되어 있으며,  
한 프로젝트에서 설치한 라이브러리는  
다른 프로젝트나 시스템 전역 환경에 영향을 주지 않는다.

---

### venv는 언제부터 사용할 수 있나?

venv는 **Python 3.3부터 표준 라이브러리로 포함**되었다.
즉, 별도의 설치가 필요 없고, Python만 설치되어 있다면 바로 사용할 수 있다.

---

### venv 생성 방법

프로젝트 디렉터리에서 다음 명령어를 실행한다.

```bash
python -m venv venv
```

이 명령어는 현재 디렉터리에 `venv`라는 가상 환경 디렉터리를 생성한다.
보통 디렉터리 이름은 관례적으로 `venv`, `.venv`, `env` 중 하나를 사용한다.

---

### venv 활성화 방법

#### Windows

```bash
venv\Scripts\activate
```

#### macOS / Linux

```bash
source venv/bin/activate
```

활성화되면 터미널 프롬프트 앞에  
가상 환경 이름이 표시된다.

```bash
(venv) $
```

이 상태에서 실행되는 Python과 pip은  
**가상 환경 내부의 것**이다.

---

### venv 비활성화 방법

가상 환경을 종료하려면 다음 명령어를 사용한다.

```bash
deactivate
```

---

### venv에서 패키지 관리

가상 환경이 활성화된 상태에서 설치한 라이브러리는  
해당 venv에만 설치된다.

```bash
pip install flask
```

설치된 패키지를 파일로 정리하려면 다음을 사용한다.

```bash
pip freeze > requirements.txt
```

다른 환경에서 동일한 라이브러리 구성을 재현할 때는:

```bash
pip install -r requirements.txt
```

---

### venv의 특징 정리

- Python 표준 라이브러리
- 가볍고 단순함
- 프로젝트 단위 환경 분리에 적합
- 별도의 도구 설치 불필요

---

### venv의 한계

venv는 가볍고 충분히 강력하지만, 몇 가지 한계도 있다.

- Python 버전 자체는 관리하지 않는다
- 데이터 과학, 머신러닝 환경에서는 다소 불편할 수 있다
- 패키지 의존성 해결은 pip에 의존한다

이러한 이유로  
보다 강력한 환경 관리가 필요한 경우 **conda**가 사용되기도 한다.
