## Flask란?

**Flask**는 Python으로 작성된 **경량 웹 프레임워크(Web Framework)**이다.  
웹 애플리케이션을 만들기 위해 필요한 최소한의 핵심 기능만 제공하며,  
나머지는 개발자가 필요한 구성 요소를 선택하여 확장하도록 설계되었다.

Flask는 흔히 **마이크로 프레임워크(Micro Framework)**로 분류된다.  
이는 기능이 부족하다는 의미가 아니라,  
**인증, ORM, 관리자 페이지와 같은 부가 기능을 코어에 포함하지 않고  
확장 형태로 제공하는 것**을 의미한다.

---

## Web Framework란?

웹 프레임워크란 **웹 애플리케이션 개발 시 반복적으로 발생하는 작업을 추상화하여  
개발자가 비즈니스 로직에 집중할 수 있도록 도와주는 도구**이다.

웹 애플리케이션을 개발할 때 일반적으로 다음과 같은 작업이 필요하다.

- HTTP 요청(Request)과 응답(Response) 처리
- URL과 처리 로직의 매핑
- HTML 템플릿 렌더링
- 폼 데이터 및 파라미터 처리
- 세션, 쿠키 관리

Flask는 이러한 기능 중 **필수적인 부분만 제공**하며,  
필요한 기능은 개발자가 라이브러리 형태로 직접 선택하여 구성한다.

---

## Flask의 특징

### 1. 경량 구조와 빠른 시작

- Flask는 매우 가볍고 단순한 구조를 가진다.
- 복잡한 설정 없이 바로 실행이 가능하다.
- 작은 코드로도 웹 애플리케이션의 전체 흐름을 이해할 수 있다.

```python
from flask import Flask

app = Flask(__name__)

@app.route("/")
def index():
    return "Hello, Flask"

if __name__ == "__main__":
    app.run()
```

※ `app.run()`으로 실행되는 서버는 개발용 서버이며,  
실제 운영 환경에서는 Gunicorn, uWSGI와 같은 WSGI 서버와 함께 사용된다.

---

### 2. 높은 자유도와 명시적인 설계

- Flask는 프로젝트 구조를 강제하지 않는다.
- 개발자가 직접 아키텍처와 디렉터리 구조를 설계해야 한다.
- 이는 곧 **설계 책임이 개발자에게 있다는 의미**이기도 하다.

이러한 특성 때문에 Flask는 개인 프로젝트, 사내 관리 도구, API 서버에서 특히 많이 사용된다.

---

### 3. 확장 중심의 생태계

Flask 자체는 최소 기능만 제공하지만,  
공식 및 서드파티 확장 라이브러리를 통해 기능을 확장할 수 있다.

대표적인 확장 라이브러리는 다음과 같다.

- Flask-SQLAlchemy : ORM 기반 DB 연동
- Flask-Migrate : 데이터베이스 마이그레이션
- Flask-Login : 인증 및 세션 관리
- Flask-WTF : 폼 및 입력 검증
- Flask-RESTful / Flask-API : REST API 구성

이러한 확장을 통해 **중규모 서비스까지 안정적으로 구성할 수 있다**.

---

## Flask의 장점

### 1. 빠른 개발 속도

- 설정 부담이 적다.
- 간단한 API 서버나 내부 도구를 빠르게 구현할 수 있다.
- 프로토타이핑에 매우 적합하다.

### 2. 높은 유연성

- 아키텍처를 직접 설계할 수 있다.
- REST API, 백엔드 서버, 마이크로서비스 구성에 적합하다.
- 불필요한 기능을 포함하지 않는다.

### 3. Python 생태계와의 궁합

- 데이터 분석, AI, 자동화 도구와 쉽게 연동된다.
- Python 기반 서비스의 백엔드로 활용하기 좋다.

---

## Flask의 단점

### 1. 기본 제공 기능 부족

- 인증, 관리자 페이지, ORM 등이 기본 제공되지 않는다.
- 초기 설계 단계에서 선택해야 할 요소가 많다.

### 2. 설계 역량에 따른 품질 편차

- 자유도가 높은 만큼 구조가 제각각이 될 수 있다.
- 팀 프로젝트에서는 코드 규칙과 구조 합의가 필수적이다.

### 3. 대규모 서비스에서는 추가 고려 필요

- 대규모 트래픽, 복잡한 도메인 구조에서는  
  Django나 Spring 같은 프레임워크가 더 적합할 수 있다.
- Flask로도 가능하지만, 설계 난이도가 높아진다.
