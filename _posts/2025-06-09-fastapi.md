## FastAPI란?

**FastAPI**는 Python으로 작성된 **고성능 웹 프레임워크(Web Framework)**이다.  
REST API 서버 구축에 최적화되어 있으며,  
타입 힌트(Type Hint)를 적극 활용하여 **빠른 개발과 높은 안정성**을 동시에 제공한다.

FastAPI는 처음부터 **API 서버 구축을 주 목적**으로 설계된 프레임워크로 
JSON 기반 통신과 비동기 처리를 기본 전제로 한다.

---

## Web Framework란?

웹 프레임워크란 **웹 애플리케이션 개발 시 반복적으로 발생하는 작업을 추상화하여  
개발자가 비즈니스 로직에 집중할 수 있도록 도와주는 도구**이다.

웹 애플리케이션을 개발할 때 일반적으로 다음과 같은 작업이 필요하다.

- HTTP 요청(Request)과 응답(Response) 처리
- URL과 처리 로직의 매핑
- 요청 데이터 검증 및 파라미터 처리
- 응답 데이터 직렬화
- 인증, 에러 처리

FastAPI는 이러한 작업 중에서도  
**API 서버에 필요한 요소를 중심으로 기능을 제공**한다.

---

## FastAPI의 특징

### 1. API 서버에 특화된 설계

- FastAPI는 HTML 렌더링보다 **REST API 구축**에 초점을 맞춘 프레임워크이다.
- 요청과 응답은 기본적으로 JSON 형태로 처리된다.
- OpenAPI 규격을 기본 지원한다.

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"message": "Hello, FastAPI"}
```


---

### 2. 타입 힌트 기반의 데이터 검증

- Python 타입 힌트를 적극 활용한다.
- 요청 데이터 검증을 자동으로 처리한다.
- 잘못된 요청은 자동으로 400 에러를 반환한다.

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/items/{item_id}")
def read_item(item_id: int):
    return {"item_id": item_id}

```

이러한 방식으로 **코드 수준에서 API 명세를 정의**할 수 있다.

---

### 3. 자동 API 문서 제공

FastAPI는 애플리케이션 실행 시  
다음과 같은 **API 문서를 자동으로 제공**한다.

- Swagger UI
- ReDoc

별도의 설정 없이도  
API 테스트와 명세 공유가 가능하다.

---

### 4. 비동기(Async) 처리 지원

- FastAPI는 **ASGI(Asynchronous Server Gateway Interface)** 기반으로 동작한다.
- 비동기 처리를 기본으로 지원한다.
- 동시 요청이 많은 환경에 유리하다.

```python
@app.get("/async")
async def async_example():
    return {"message": "async response"}

```

실제 운영 환경에서는 Uvicorn, Hypercorn과 같은 ASGI 서버와 함께 사용된다.

## FastAPI의 장점

### 1. 높은 성능

- Starlette(스타얼랏) 기반으로 매우 빠른 성능을 제공한다.
- Python 웹 프레임워크 중 상위권 성능을 가진다.

### 2. 명확한 API 구조

- 타입 힌트를 통한 명시적인 API 정의
- 자동 문서화로 명세 관리가 용이하다.
- 프론트엔드·백엔드 협업에 유리하다.

### 3. 비동기 처리에 적합

- I/O 작업이 많은 서비스에 적합하다.
- 실시간 처리, 대규모 트래픽 대응에 유리하다.

---

## FastAPI의 단점

### 1. 웹 페이지 렌더링에는 부적합

- 서버 사이드 HTML 렌더링에는 적합하지 않다.
- 전통적인 웹 애플리케이션보다는 API 서버에 초점이 맞춰져 있다.

### 2. 타입 힌트에 대한 이해 필요

- Python 타입 힌트에 익숙하지 않으면 진입 장벽이 될 수 있다.
- 초보자에게는 코드가 다소 복잡하게 느껴질 수 있다.

### 3. 범용 웹 프레임워크로는 제약 존재

- Flask나 Django처럼 범용 웹 기능을 모두 제공하지 않는다.
- 목적이 명확하지 않은 프로젝트에는 과한 선택일 수 있다.
